# 設計レビュースキル

## 目的

設計書と要件定義ドキュメントを照合し、実装フェーズに入る前に誤り・過不足・矛盾・曖昧さを検出する。
このスキルはオーケストレーション(対象特定 → エージェント呼び出し → 結果ハンドリング)を担当し、実際のレビュー分析は **design_reviewer エージェント**に委譲する。

## 手順

### 1. 対象ドキュメントの特定

- ユーザーが指定した機能名・ドキュメント名から対象ファイルを特定する
- 要件定義(`docs/requirements/`)と設計書(`docs/design/`)の存在を確認する

#### ドキュメントが揃わない場合

- **要件定義が存在しない場合** → 設計レビューは実施せず、先に `/requirement` で要件定義を作成することを提案する
- **設計書が存在しない場合** → 設計レビューは実施せず、先に `/design` で設計書を作成することを提案する
- **両方存在する場合** → 手順 2 に進む

### 2. design_reviewer エージェントの呼び出し

`design_reviewer` エージェント(`.claude/agents/design_reviewer.md`)を Agent ツール(subagent_type: `design_reviewer`)で呼び出す。プロンプトには以下を含める:

```
以下のドキュメントについて設計レビューを実施してください。

- 要件定義: docs/requirements/YYYYMMDD-xxx.md
- 設計書: docs/design/YYYYMMDD-xxx.md
- 共通規約: CLAUDE.md / AGENTS.md
- 現状の設計: docs/design/current-state.md(存在する場合)
```

### 3. 結果のハンドリング

エージェントから返されたレビュー結果をユーザーに提示し、次のステップを案内する:

- **Request Changes** の場合 → 要件定義・設計書の修正後、再度 `/design_review` を実行するよう案内する
- **Approve** の場合 → `/implementation` の実行を案内する

## 注意事項

- ドキュメントの編集は行わない(指摘と修正案の提示のみ)
- レビューの分析ロジック・観点・出力フォーマットは design_reviewer エージェント側で定義されている
