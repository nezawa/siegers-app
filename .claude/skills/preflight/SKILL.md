# コミット前チェック

変更をコミット・push する前に以下を順に実行する:

```bash
npm run lint
npm run build
```

- どちらかが失敗したら、エラーを修正してから再実行する。失敗したままコミットしない。
- `npm run build` は本番ビルド(Vercel デプロイと同条件)。型エラーもここで検出される。
- 成績テーブル系の変更(列の追加・幅調整・sticky 列)をした場合は、
  ビルドだけでなく dev サーバーで実際の表示(特にモバイル幅での横スクロール)も確認するのが望ましい。
