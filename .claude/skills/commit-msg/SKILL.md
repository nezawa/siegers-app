# コミットメッセージの書き方

このリポジトリのコミットメッセージは **日本語・1行・簡潔** が流儀。
Conventional Commits(`feat:` `fix:` など)の接頭辞は使わない。

## 実例(既存の履歴より)

```
規定を追加
試合詳細の列の幅調整
先攻後攻のjson,裏の×の修正
成績画面スクロールに対応
成績で期間のソートができるように変更
```

## ルール

- 「何をしたか」を体言止めまたは「〜に対応」「〜を修正」「〜を追加」で短く書く
- 対象領域(成績画面、試合詳細、json など)を先頭に置くと分かりやすい
- 1コミット1目的。無関係な変更が混ざっていたら分割を提案する
- コミット前に `git diff --staged` で内容を確認し、メッセージと一致しているか確かめる
- コミットは main ブランチに直接行う運用(PRフローは使っていない)
