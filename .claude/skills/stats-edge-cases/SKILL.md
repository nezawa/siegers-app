# 成績計算の境界値チェックリスト

率系指標はゼロ除算や表記の慣習があり、壊れても気づきにくい。
集計ロジック(計算式の本体は `src/lib/stats.ts`。利用側は `src/app/(pages)/players/page.tsx`、`src/app/(pages)/players/[id]/page.tsx`、`src/app/(pages)/games/[id]/page.tsx`)を変更したら以下を確認する。
`lib/stats.ts` を直すと成績一覧と個人ページの両方に効くので、片方だけ直す修正は原則しない。

## 打撃

- [ ] **0打数の選手** — 打率・長打率が `NaN` / `Infinity` にならず `---` 等で表示されるか
- [ ] **打席はあるが打数0**(四球のみ等)— 出塁率は計算でき、打率は表示できないケース
- [ ] **規定打席未満の選手** — ランキングや並び順から除外/区別されるか(規定値は `settings` テーブルの `qualified_pa`)
- [ ] **打撃成績が1試合もない選手** — 通算成績画面でエラーにならないか

## 投手

- [ ] **投球回0** — 防御率・奪三振率が `Infinity` にならないか(0回で失点だけあるケースも)
- [ ] **投球回の端数** — `ip` の 1/3・2/3 回の扱い。表示(例: 5.1 = 5回1/3)と計算(5.333...)で値が異なる慣習に注意。既存の変換処理を grep して同じ方式を使う
- [ ] **規定投球回未満** — `settings.qualified_ip` による除外/区別が効いているか
- [ ] **投手成績が1試合もない選手** — `is_pitcher` フラグとの整合

## 集計・期間フィルタ

- [ ] **期間フィルタで該当試合が0件** — 空のテーブルが崩れず表示されるか
- [ ] **game_type**(公式戦/練習試合)の絞り込みと通算値の整合
- [ ] **引き分け・コールド等の特殊な試合** — 勝敗数の合計が試合数と一致するか(`result` が null の試合)
- [ ] **同姓同名・背番号 null の選手** — 表示とソートが壊れないか

## 確認方法

可能なら dev サーバーで実データを表示して目視確認する。
とくに `/players`(通算)と `/players/[id]`(個人)の両方で同じ選手の値が一致することを確認する。
