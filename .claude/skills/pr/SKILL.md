# PR 作成スキル

## 目的

現在のブランチの変更内容(diff)を分析し、`.github/PULL_REQUEST_TEMPLATE.md` のフォーマットに沿った PR 本文を生成し、`gh` コマンドで GitHub に PR を作成する。

## 前提知識

- PR テンプレート: `.github/PULL_REQUEST_TEMPLATE.md`
- 要件定義: `docs/requirements/` ディレクトリ
- 設計書: `docs/design/` ディレクトリ

## 手順

### 1. ブランチ・差分の確認

- 現在のブランチ名を確認する(`git branch --show-current`)
- マージ先ブランチとの差分を取得する(`git diff main --stat` および `git diff main`)
- コミット履歴を確認する(`git log main..HEAD --oneline`)
- `main` 上で作業している場合は、先に作業ブランチの作成を促す

### 2. 変更内容の分析

- 変更ファイルを分類する(フロント / 成績ロジック / DB / API / インフラ / ドキュメント / 設定)
- 各変更の目的と内容を要約する
- 対応する要件定義・設計書があれば紐付ける

### 3. PR 本文の生成

`.github/PULL_REQUEST_TEMPLATE.md` に沿って以下を埋める。

- **概要**: 変更の目的と理由を簡潔に
- **変更内容**: 具体的な変更点をリストアップ
- **影響範囲**: 変更ファイルの分類からチェックを入れる
- **チェックリスト**: 該当するものにチェックを入れる

### 4. ユーザーへの提示

- 生成した PR 本文をユーザーに提示する
- 修正の要望があれば反映する
- スクリーンショットなど、コードから判断できない項目はユーザーに確認する

### 5. GitHub PR の作成

- 未コミットの変更がないか `git status` で確認し、ある場合はコミットを促す(コミットメッセージは `/commit-msg` に従う)
- ユーザーの承認後、`gh pr create` コマンドで PR を作成する

```
gh pr create --base main --head [ブランチ名] --title "[タイトル]" --body '[PR本文]'
```

- 作成された PR の URL をユーザーに報告する
- `gh` コマンドが失敗した場合はエラー内容を報告し、手動での対応を案内する

## 注意事項

- 差分から読み取れる事実のみ記載する(推測で補完しない)
- テスト実施状況・動作確認はユーザーに確認する(勝手にチェックを入れない)
- PR 本文は日本語で作成する
